# Hospital Management Technology Platform (HMTP)

A scalable, cloud-native hospital management system designed for multi-hospital environments. Features patient records, appointments, billing, insurance claims, role-based access control, and secure API-driven integrations.

---

## Features

### Backend (FastAPI)

- **Authentication & RBAC** — JWT login/register, roles (admin, doctor, patient, nurse, lab_technician, pharmacist)
- **Patient Management** — CRUD, demographics, emergency contacts
- **Doctor Management** — Profiles, availability slots, specialization
- **Appointments** — Booking with doctor availability and conflict checks
- **Pharmacy** — Medicines, prescriptions, dispensing, stock updates
- **Laboratory** — Lab tests, results, patient lab history
- **Billing & Payments** — Invoices, payments, insurance claims
- **Procedures** — Medical procedure records per patient
- **Audit & Compliance** — Immutable audit logs for sensitive actions
- **Multi-tenancy** — Hospital-scoped data via `hospital_id` (JWT + middleware)

### Frontend (Next.js)

- **Landing** — Hero, platform intro, navigation
- **Login** — Patient/staff login with API integration
- **Dashboard** — Role-based dashboards (admin, doctor, patient)
- **Departments** — Department listing and info
- **Doctors** — Doctor profiles and availability
- **Appointments** — Appointment booking and list
- **Services** — Hospital services overview
- **Contact** — Contact information
- **Shared UI** — Header, Footer, Tailwind styling

### Other Frontends

- **hmtp-project** — Vite + React + MUI client (patient/doctor/admin dashboards, DataGrid, Recharts). Generate with `python generate_hmtp_project.py`.
- **frontend_phase6** — Legacy React app (generated by `frontend/generate_frontend.py`).

## Getting Started

### Prerequisites

- **Docker & Docker Compose** (for full stack)
- **Node.js 18+** (for local frontend)
- **Python 3.11+** (for local backend)

### Run with Docker (recommended)

```bash
docker compose up --build
```

- Frontend: http://localhost:3000  
- Backend API: http://localhost:8000  
- API docs: http://localhost:8000/api/v1/docs  

### Run backend only (local)

```bash
cd backend
pip install -r requirements.txt
# Optional: set DATABASE_URL or use default SQLite
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Run frontend only (local)

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:3000.

### Seed sample patients (backend)

```bash
cd backend
python seed_patients.py
```
